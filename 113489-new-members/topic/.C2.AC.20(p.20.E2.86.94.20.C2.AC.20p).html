1) It can be done!

It's easy to do it classically, e.g.

variable (p : Prop)

example : ¬ (p ↔ ¬ p) :=
begin
  -- use excluded middle
  cases (classical.em p),
  { simp [h]},
  { simp [h]}
end

But it can also be done constructively. In the next post below, there's a hint. Don't scroll down/unfold if you don't want to see it.

variable (p : Prop)

example : ¬ (p ↔ ¬ p) :=
begin
  -- let's assume p ↔ ¬ p and prove false
  intro h,
  -- I claim that ¬ p is true.
  have hnp : ¬ p,
  { -- indeed if we assume p,
    intro hp,
    -- then we can deduce ¬ p
    have hnp : ¬ p,
      rw ←h,
      assumption,
    -- and now we have proofs of p and ¬ p so we can get a contradiction
    contradiction,
  },
  -- but now we know ¬ p is true, so we can deduce p,
  have hp : p,
  { rw h,
    assumption},
  -- and now again p and ¬ p are both proved, which is our contradiction
  contradiction
end

Patrick Stevens (Apr 16 2020 at 12:11):

By way of a random aside, the same in Agda (the term was generated automatically by the automatic proof search).

The important first step is to note that if we want to produce a False, we almost certainly got it by constructing two Ps and using the P -> (P -> False) direction of the iff. There's ultimately no other way to produce a False.

{-# OPTIONS --safe --warning=error --without-K #-}

module Logic.RandomBits where

record False : Set where

record Iff (A : Set) (B : Set) : Set where
  field
    to : A → B
    from : B → A

Not : (A : Set) → Set
Not A = A → False

thing : (P : Set) → Iff P (Not P) → False
thing P record { to = to ; from = from } = to (from (λ z → to z z)) (from (λ z → to z z))
